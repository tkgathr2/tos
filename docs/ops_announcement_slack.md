### S-5 は履歴フェーズ

- S-5 は 2025-12-14 に closeout 完了
- S-5 への変更は禁止
- S-5 ファイルは参照のみ可能

### S-7 は不要（確定）

- S-7 は 2025-12-14 に「やらない」と確定
- 自動リトライは S-6 で既に実装済み
- 残りの機能は外部ツール/手動運用で代替可能
- 再検討は 2025-06-14 以降のみ許可

```
【TOS 質問】

■ 質問カテゴリ
□ 操作方法
□ エラー対応
□ 仕様確認
□ その他

■ 質問内容
（具体的な質問を記載）

■ 試したこと
（既に試した操作があれば記載）

■ 参照したドキュメント
（確認したドキュメントがあれば記載）

■ 環境情報（該当する場合）
- commit hash:
- 実行したコマンド:
- エラーメッセージ:
```

## 障害報告への導線

障害が発生した場合は、以下の手順で報告してください。

1. **ログ収集**: 障害時のログセットを収集
2. **テンプレート使用**: [failure_report_template.md](failure_report_template.md) に沿って記入
3. **窓口連絡**: 運用窓口へ報告

詳細な手順は [s6_routines.md#障害対応ルーチン](s6_routines.md#障害対応ルーチン) を参照。
