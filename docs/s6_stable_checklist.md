# S-6 Stable 運用チェックリスト

## 概要

本チェックリストは S-6 stable 運用を開始・継続するために必要な項目を網羅しています。

## 使用方法

1. 運用開始前に各項目を順番に確認する
2. 確認が完了したら `[ ]` を `[x]` に変更する
3. すべての項目が `[x]` になったら運用開始可能
4. 定期的に再確認を行う

---

## A. 運用開始前チェック

### 環境確認

- [ ] Python 3.12+ がインストールされている
- [ ] PowerShell が利用可能
- [ ] OPENAI_API_KEY が環境変数に設定されている
- [ ] ANTHROPIC_API_KEY が環境変数に設定されている
- [ ] 作業ディレクトリが C:\Users\takag\00_dev\tos である

### ドキュメント確認

- [ ] README.md を読んだ
- [ ] docs/s6_stable.md を読んだ
- [ ] docs/runbook.md を読んだ
- [ ] S-6 運用ルールを理解した

### 前提条件確認

- [ ] S-5 closeout が完了している
- [ ] 正史コンテキスト v1.0 に準拠している
- [ ] git リポジトリが最新状態である

---

## B. 実行前チェック

### checkpoint 確認

- [ ] checkpoint を作成した
- [ ] logs/last_checkpoint.txt が存在する

### 入力確認

- [ ] job_input.json が配置されている（必要な場合）
- [ ] config_v0_3.json の設定が正しい

### 状態確認

- [ ] workspace/ 配下の状態を確認した
- [ ] logs/ 配下の状態を確認した

---

## C. 実行後チェック

### 正常終了確認

- [ ] test モードが成功した
- [ ] phase が期待通りである（done または job_loop_complete）
- [ ] job_result.json が生成された（job_loop_complete の場合）

### ログ確認

- [ ] logs/steps/step_*.json が生成された
- [ ] logs/last_run_summary.json が生成された
- [ ] エラーログがない

---

## D. 定期確認

### 週次確認

- [ ] API キーの有効期限を確認
- [ ] ログファイルのサイズを確認
- [ ] git リポジトリの状態を確認

### 問題発生時確認

- [ ] エラーメッセージを記録した
- [ ] ログをバックアップした
- [ ] rollback を実行した
- [ ] test モードで状態確認した

---

## E. S-6 運用ルール確認

### 許可事項

- [ ] ドキュメント修正は許可されている
- [ ] ログ文言修正は許可されている
- [ ] コメント追加は許可されている
- [ ] 軽微なバグ修正は承認後に許可されている

### 禁止事項

- [ ] 新機能追加は禁止されている
- [ ] 仕様変更は禁止されている
- [ ] 設定スキーマ変更は禁止されている
- [ ] 破壊的変更は禁止されている

---

- 不明点がある場合は docs/runbook.md を参照
